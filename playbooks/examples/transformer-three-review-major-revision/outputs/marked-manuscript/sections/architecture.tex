\section{Model Architecture}

The Transformer follows the familiar encoder-decoder layout. The encoder maps an input sequence $(x_1, \ldots, x_n)$ to contextualized continuous representations, and the decoder generates the output sequence auto-regressively from those representations.

Each encoder layer contains a multi-head self-attention sub-layer followed by a position-wise feed-forward network. The decoder mirrors this pattern and adds encoder-decoder attention between its self-attention block and feed-forward block. Residual connections and layer normalization are applied throughout.

\subsection{Scaled Dot-Product Attention}

Given queries $Q$, keys $K$, and values $V$, we compute attention as
\[
\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V.
\]
This formulation is efficient on GPUs and allows the model to relate all positions in a sequence without recurrence.

\subsection{Multi-Head Attention and Positional Signals}

Instead of using a single attention map, we project the inputs into multiple heads, compute attention in parallel, concatenate the resulting values, and apply a final linear projection. Positional information is injected by adding sinusoidal signals to the token embeddings at the bottoms of the encoder and decoder stacks.
